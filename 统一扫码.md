核心文件解析
一、应用级入口与配置
表格
文件	功能简述
entryability/EntryAbility.ets	应用入口。继承自 UIAbility，管理应用生命周期，存储全局上下文，动态申请相机等权限，并配置窗口属性。
AppScope/app.json5	应用全局配置。定义包名、版本号、图标、标签，以及扫码直达服务所需的域名配置。
entry/src/main/module.json5	模块配置。声明 Ability、页面路由、所需权限（如 ohos.permission.CAMERA）及设备类型。
二、导航与公共模块
支撑页面跳转、全局工具方法和 UI 基础组件。
表格
文件	功能简述
pages/Index.ets	主页面 / 功能导航。汇聚所有功能入口，用户点击后跳转到默认扫码、自定义扫码、图像识码、码图生成等子页面。
resources/base/profile/main_pages.json	页面路由表。定义应用中所有页面的路径，是系统页面跳转的依据。
common/Utils.ets	通用工具类。封装屏幕尺寸获取、像素转换、设备信息等公共方法。
common/Logger.ts	日志工具。统一封装 hilog，便于调试和问题追踪。
common/PermissionsUtil.ets	权限工具。检查并动态申请运行时权限（如相机权限）。
三、自定义界面扫码（核心模块）
自定义扫码是示例中最复杂的功能，包含状态管理 V1 / V2 两种实现。
3.1 入口与主页面
表格
文件	功能简述
customScan/CustomPage.ets	自定义扫码功能入口，提供进入扫码界面的按钮。
customScan/pages/ScanPage.ets	扫码主页面。组合标题、扫描框、底部栏等所有 UI 组件，处理交互逻辑。
customScanV2/pages/ScanPage.ets	同上，使用状态管理 V2 实现。
3.2 业务与布局核心
表格
文件	功能简述
customScan/model/ScanService.ets	扫码核心服务。封装 @kit.ScanKit SDK 交互，包括初始化、预览、闪光灯、变焦、结果回调。
customScan/model/ScanLayout.ets	界面布局管理。计算扫码框位置、大小、扫描线动画、识别区域。
customScanV2/model/*	V2 版核心服务，逻辑与 V1 一致。

窗口管理、设备信息、XComponent 生命周期

V2 新增：ConfigStorage 跨组件状态共享
四、其他功能模块
表格

核心技术点总结
1. ScanKit 多场景 API
表格
场景	核心 API	特点
默认界面扫码	startScanForResult()	快速集成
自定义界面扫码	init/start/stop/release	高度可控、支持闪光灯 / 变焦
图像识码	decode()	支持图片、相机流识别
码图生成	createBarcode()	生成二维码 / 条形码并展示
2. 状态管理 V1 vs V2
V1：@State/@Link/@Provide
V2：@ObservedV2/@Trace/ConfigStorage（性能更优）
3. 相机与权限
动态相机权限申请
相机生命周期严格管理，避免资源泄漏
4. 设备与窗口适配
屏幕常亮
多设备自适应扫码框布局
5. 系统交互能力
扫码成功提示音
页面路由管理
扫码直达（Deep Link）
相册选取、实时相机流解析
总结
本项目是 HarmonyOS ScanKit 生产级完整示例，覆盖：
默认扫码
自定义扫码
图像识码
码图生成
扫码直达服务
