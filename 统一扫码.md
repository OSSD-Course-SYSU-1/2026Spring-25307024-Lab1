##### 核心文件解析

\## 一、应用级入口与配置

| 文件 | 功能简述 |

| :--- | :--- |

| `entryability/EntryAbility.ets` | \*\*应用入口\*\*。继承自 `UIAbility`，管理应用生命周期，存储全局上下文，动态申请相机等权限，并配置窗口属性。 |

| `AppScope/app.json5` | \*\*应用全局配置\*\*。定义包名、版本号、图标、标签，以及扫码直达服务所需的域名配置。 |

| `entry/src/main/module.json5` | \*\*模块配置\*\*。声明 Ability、页面路由、所需权限（如 `ohos.permission.CAMERA` 等）及设备类型。 |



\## 二、导航与公共模块



支撑页面跳转、全局工具方法和UI基础组件。



| 文件 | 功能简述 |

| :--- | :--- |

| `pages/Index.ets` | \*\*主页面/功能导航\*\*。汇聚所有功能入口，用户点击后跳转到默认扫码、自定义扫码、图像识码、码图生成等子页面。 |

| `resources/base/profile/main\\\_pages.json` | \*\*页面路由表\*\*。定义应用中所有页面的路径，是系统页面跳转的依据。 |

| `common/Utils.ets` | \*\*通用工具类\*\*。封装屏幕尺寸获取、像素转换、设备信息等公共方法。 |

| `common/Logger.ts` | \*\*日志工具\*\*。统一封装 `hilog`，便于调试和问题追踪。 |

| `common/PermissionsUtil.ets` | \*\*权限工具\*\*。检查并动态申请运行时权限（如相机权限）。 |



\## 三、自定义界面扫码（核心模块）



自定义扫码是示例中最复杂的功能，使用了状态管理 V1（`customScan`）和 V2（`customScanV2`）两种实现。以下是核心文件的解析。



\### 3.1 入口与主页面



| 文件 | 功能简述 |

| :--- | :--- |

| `customScan/CustomPage.ets` | 自定义扫码功能的入口，提供进入扫码界面的按钮。 |

| `customScan/pages/ScanPage.ets` | \*\*扫码主页面\*\*。组合标题、扫描框、底部栏等所有UI组件，处理各组件间的交互逻辑。V2 版中部分状态通过 `ConfigStorage` 共享。 |

| `customScanV2/pages/ScanPage.ets` | 同上，使用状态管理 V2 实现的主页面。 |



\### 3.2 业务与布局核心



| 文件 | 功能简述 |

| :--- | :--- |

| `customScan/model/ScanService.ets` | \*\*扫码核心服务\*\*。封装与 `@kit.ScanKit` SDK 的全部交互，包括初始化、启停预览、闪光灯/变焦控制、扫码结果回调等。 |

| `customScan/model/ScanLayout.ets` | \*\*界面布局管理\*\*。根据屏幕尺寸计算扫码框位置和大小，控制扫描线的动画，并设定有效的识别区域（`visionRect`）。 |

| `customScanV2/model/ScanService.ets` | V2 版扫码核心服务，逻辑与 V1 基本一致。 |

| `customScanV2/model/ScanLayout.ets` | V2 版布局管理，同样负责扫码框和动画。 |

\## 四、其他功能模块



这些模块相对独立，核心逻辑通常集中在少数文件中。



| 模块 | 核心文件 | 功能简述 |

| :--- | :--- | :--- |

| \*\*默认界面扫码\*\* | `pages/defaultScan/DefaultScan.ets` | 调用 `scanCore.startScanForResult()`，直接唤起系统级统一扫码界面，处理返回结果。集成最简单。 |

| \*\*扫码直达服务\*\* | `pages/access/ScanAccess.ets` <br> `pages/access/ScanDetail.ets` | 处理通过控制中心扫描二维码唤醒应用后的跳转逻辑，展示直达服务详情页。 |

| \*\*图像识码（本地图片）\*\* | `pages/detectBarcode/DecodeBarcode.ets` | 从系统相册选择图片后，调用 `detectBarcode.decode()` 识别图片中的码。 |

| \*\*图像识码（预览流）\*\* | `pages/detectBarcode/DecodeCameraYuv.ets` | 打开相机，实时获取 YUV 图像帧并送入识码接口，实现连续识码。 |

| \*\*码图生成\*\* | `pages/generateBarcode/CreateBarcode.ets` | 使用 `generateBarcode.createBarcode()` 将字符串或字节数组生成自定义格式的码图（PixelMap）。 |

| \*\*结果展示\*\* | `pages/resultPage/ResultPage.ets` | 统一的扫码/识码结果展示页，显示码内容、类型等信息。 |

##### 核心技术点总结

\# ScanKit 示例项目 – 核心技术点总结



本项目通过完整的实操示例，展示了 HarmonyOS 统一扫码服务 (Scan Kit) 的多种应用方式，其核心技术点涵盖以下几个方面：



\---



\## 1. Scan Kit 多场景 API 的应用



项目覆盖了 Scan Kit 提供的四大主要功能，并展示了其 API 的典型用法：



| 技术场景 | 核心 API | 关键技术点 |

| :--- | :--- | :--- |

| \*\*默认界面扫码\*\* | `scanCore.startScanForResult()` | 一行代码唤起系统级扫码界面，无需额外权限处理，适用于快速集成。 |

| \*\*自定义界面扫码\*\* | `customScan.init()` / `start()` / `stop()` / `release()` | 需自行管理相机权限、`XComponent` 渲染、扫码生命周期；支持动态变焦、闪光灯控制、识别区域设定等高级功能。 |

| \*\*图像识码\*\* | `detectBarcode.decode()` | 支持从相册图片或实时相机帧数据（YUV）中检测码图，灵活处理静态图像与动态视频流。 |

| \*\*码图生成\*\* | `generateBarcode.createBarcode()` | 将字符串或字节数组生成指定格式（如二维码、条形码）的 `PixelMap`，可直接在 `Image` 组件上展示。 |



\---



\## 2. 状态管理版本演进



项目同时提供了两种状态管理的实现，便于开发者对比和选型：



\- \*\*状态管理 V1 (`customScan`)\*\*

&#x20; - 基于 `@State`、`@Link`、`@Provide/Consume` 等装饰器，实现组件内和跨组件状态传递。

&#x20; - 适用于已有项目或熟悉传统状态管理模式的开发者。



\- \*\*状态管理 V2 (`customScanV2`)\*\*

&#x20; - 采用 `@ObservedV2`、`@Trace`、`ConfigStorage` 等新特性，实现更细粒度的状态变更和性能优化。

&#x20; - 展示了推荐的现代 ArkUI 开发方式，适合新项目或追求高性能的场景。



\---



\## 3. 相机与权限处理



\- \*\*动态权限申请\*\*

&#x20; 使用 `PermissionsUtil` 对敏感权限（如 `ohos.permission.CAMERA`）进行运行时检查和申请，符合安全规范。



\- \*\*相机生命周期绑定\*\*

&#x20; 自定义扫码页面通过 `XComponentService` 管理 `XComponent` 的创建与销毁，并与页面生命周期（`onPageShow`/`onPageHide`）严格对齐，做到\*\*即用即开，即离即关\*\*，防止资源泄漏。



\---



\## 4. 窗口与设备适配



\- \*\*屏幕常亮控制\*\*

&#x20; 通过 `WindowService` 设置窗口属性，在扫码过程中保持屏幕常亮，优化用户体验。



\- \*\*自适应布局计算\*\*

&#x20; `ScanLayout` 依据设备屏幕尺寸，动态计算扫码框位置、大小以及识别有效区域，确保在不同设备上都有良好的显示效果。



\---



\## 5. 公共事件与系统交互



\- \*\*公共事件订阅\*\*

&#x20; 通过 `CommonEventManager` 订阅屏幕方向变化、设备配置变更等系统事件，及时刷新 UI 或重新计算布局。



\- \*\*音效与振动反馈\*\*

&#x20; 扫码成功后播放提示音（`PromptTone`），并可结合振动权限（`ohos.permission.VIBRATE`）提供触觉反馈，提升交互质感。



\---



\## 6. 路由与生命周期管理



\- \*\*页面路由\*\*

&#x20; 使用 `main\\\_pages.json` 定义页面路径，结合 `@ohos.router` 实现页面跳转，符合 Stage 模型规范。



\- \*\*UIContext 传递\*\*

&#x20; 通过 `UIContextSelf` 管理 `UIAbility` 的上下文，确保在非页面组件中也能安全访问 UI 操作（如弹窗）。



\---



\## 7. “扫码直达”服务集成（Deep Link）



\- 通过 `app.json5` 和 `module.json5` 配置域名，使应用能够响应系统控制中心的扫码事件。

\- 在 `ScanAccess.ets` 中解析唤起参数（如 URL），并直达对应服务页面，实现“一扫即达”的无缝体验。



\---



\## 8. 图像数据流处理



\- \*\*本地图片选码\*\*

&#x20; 调用系统 `Picker` 获取图片，转为 `PixelMap` 后传入 `detectBarcode.decode()`。



\- \*\*相机实时流识码\*\*

&#x20; 通过 `camera` 模块获取 YUV 格式的预览帧，持续送入识码接口，实现 \*\*实时识码\*\* 效果，类似扫一扫功能。



\---



\## 总结



本示例代码全面覆盖了 HarmonyOS Scan Kit 的核心使用场景和技术要点，从简单调用到深度定制，从权限管理到 UI 构建，为开发者提供了一个\*\*生产级\*\*的参考实现。无论是快速集成还是深度二次开发，均可从中获得清晰的指导。
