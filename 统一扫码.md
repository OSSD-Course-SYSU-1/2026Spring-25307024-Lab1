## 核心文件解析

### 一、应用级入口与全局配置
| 文件路径 | 功能详细说明 |
| :--- | :--- |
| `entryability/EntryAbility.ets` | **应用根入口**，继承系统`UIAbility`，管理APP前后台生命周期、全局上下文、窗口初始化、运行时权限预校验 |
| `AppScope/app.json5` | 应用全局基础配置，定义包名、版本、图标名称、扫码直达域名与DeepLink唤醒规则 |
| `module.json5` | 模块能力清单，声明页面路由、相机/震动等系统权限、设备适配类型、硬件能力声明 |

---

### 二、导航与全局公共基础模块
| 文件路径 | 功能详细说明 |
| :--- | :--- |
| `pages/Index.ets` | 项目首页导航中心，聚合所有扫码功能入口，实现全页面统一跳转 |
| `main_pages.json` | 系统页面路由注册表，ArkUI页面跳转的底层路径依据 |
| `common/Utils.ets` | 通用工具合集，封装屏幕适配、像素单位转换、设备信息获取等高频方法 |
| `common/Logger.ts` | 分级日志打印工具，规范调试输出，方便线上问题排查 |
| `common/PermissionsUtil.ets` | 运行时权限封装，一键检测+申请相机等敏感权限，兼容系统安全规范 |

---

### 三、自定义扫码（项目核心功能模块）

#### 3.1 页面入口与主逻辑
| 文件 | 核心作用 |
| :--- | :--- |
| `customScan/CustomPage.ets` | 自定义扫码功能按钮入口，引导用户进入扫码页 |
| `customScan/pages/ScanPage.ets` | 完整扫码主页面，组装所有UI组件，串联扫码全流程交互 |
| `customScanV2/pages/ScanPage.ets` | 同功能，采用ArkUI新版**状态管理V2**重构，性能与维护性更强 |

#### 3.2 业务核心与布局管理
| 文件 | 核心作用 |
| :--- | :--- |
| `ScanService.ets` | **扫码最核心服务**，封装Scan Kit SDK全部接口：初始化、启停预览、闪光灯、变焦、扫码结果回调 |
| `ScanLayout.ets` | 动态计算扫码框位置、尺寸、扫描动画轨迹、算法有效识别区域 |

#### 3.3 独立原子UI组件
统一封装扫码页所有可复用视图，实现低耦合、按需组合
- 顶部标题栏、返回导航
- 相机预览渲染容器
- 外围暗色遮罩
- 上下滚动扫描线动画
- 底部闪光灯、相册、取消功能栏
- 扫码结果点位标记、弹窗、加载动画

#### 3.4 系统辅助能力
- 屏幕常亮、窗口方向锁定
- 设备分辨率、断点适配
- 系统相册唤起、扫码成功提示音与震动反馈
- 屏幕旋转、应用前后台事件监听
- V2专属：全局跨页面响应式状态管理

### 四、其他独立功能模块
| 功能模块 | 核心页面 | 能力介绍 |
| :--- | :--- | :--- |
| 默认一键扫码 | `DefaultScan.ets` | 一行代码唤起系统原生扫码，零开发快速集成 |
| 扫码直达DeepLink | `ScanAccess/ScanDetail` | 响应外部扫码唤醒APP，直达指定业务服务页 |
| 静态图像识码 | `DecodeBarcode.ets` | 读取本地相册图片，识别图中任意二维码/条码 |
| 相机流实时识码 | `DecodeCameraYuv.ets` | 实时解析相机原始YUV画面帧，连续快速识码 |
| 码图生成 | `CreateBarcode.ets` | 输入文本，一键生成自定义尺寸、样式的二维码/条形码 |
| 结果统一展示 | `ResultPage.ets` | 标准化展示所有扫码结果，统一排版交互 |

## 项目核心技术栈总结

1. **HarmonyOS Scan Kit 全场景API接入**
覆盖系统默认扫码、完全自定义扫码、静态图解码、码图生成四大官方场景

2. **ArkUI 双版本状态管理对比**
同时提供**V1传统写法**与**V2新一代响应式写法**，方便学习选型

3. **完整权限与生命周期管控**
规范动态权限申请，相机资源精准启停，杜绝内存泄漏

4. **全设备多端适配**
自适应不同屏幕尺寸、折叠屏/平板断点布局、横竖屏兼容

5. **系统深度交互能力**
页面路由、窗口控制、全局事件订阅、媒体音效、本地相册调用

6. **工程架构设计**
采用分层解耦架构：公共工具层 → 基础模型层 → 业务页面层，代码复用率高，极易二次开发与扩展
